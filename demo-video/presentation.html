Rate Limits Tab

Know your limits before Terraform does

The #1 silent killer of Okta Terraform runs — now fully visible before you run a single plan.

  • Probes 30+ Okta API endpoints in real time — including sub-resource paths most tools miss
  • Classifies each endpoint by remaining capacity and identifies the exact bottleneck
  • Generates a ready-to-use provider.tf with tuned settings based on live data
  • Volume-aware — recommendations differ for 100 vs. 10,000 resources
  • Includes parallelism, max_retries, backoff, request_timeout and more
Key insight: Terraform's default parallelism of 10 is too aggressive for most Okta orgs. OTTO surfaces this before it causes a production incident.

Debug Tab

When Terraform fails, find the answer fast

Two AI-powered tools that turn cryptic errors and dense log files into actionable fixes.

🔴

Error Decoder

Paste any Okta API error or Terraform failure message. OTTO returns:

  • Plain-English explanation of what went wrong
  • Root cause analysis powered by Claude
  • Specific config or API changes to fix it
  • Links to relevant Okta documentation
📋

TF_LOG Analyzer

Upload a TF_LOG=DEBUG output file. OTTO:

  • Extracts per-endpoint request stats and rate-limit hits
  • Detects schema validation errors (even with no HTTP requests)
  • Shows error frequency by endpoint
  • AI root cause with actionable provider config recommendations
No more manually searching through 50,000 lines of debug output. OTTO finds the signal in the noise.
